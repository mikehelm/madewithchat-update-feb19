Recent advances in AI capability have opened a door we'd be foolish not to walk through. We've spent the past few weeks rethinking our architecture from the ground up, and the benefits will show up directly in the product you use. Here's what's changing and why.

The Shift

Newer AI models have gotten good enough that they can now handle a big chunk of what we were building ourselves. We are shifting from a design where we maintain lots of small, specialized tools (each doing one narrow job) to a design where we give the AI one powerful environment and let it use normal developer tools directly.

A Concrete Example

Instead of us building and maintaining a "virtual filesystem" inside our product, we can put the agent in a remote dev machine and simply have it use Git the same way a developer would.

We spent time researching how to lean on existing, proven developer workflows and abstractions (like Git and real execution environments) given the current capability level of the models.

Architecture shift: from many custom tools to one integrated dev environment

What We Have Now

  • A working prototype of a Git-native suggestion flow that can propose changes as real Git diffs/commits, connected to a sandbox so it can run AI-written code and validate results.
  • Completed part of the development automation: the cc-verify project plus our SKILL framework. This lets us test the voice layer by creating test scenarios on the fly and running them through automation.

What It Means in Everyday Terms

  • The models got smarter, so we can stop building some internal infrastructure that was only needed because the AI used to be weaker.
  • Instead of many tiny tools, we can give the AI one realistic workspace (a remote machine) where it can do "normal dev stuff" like git status, git diff, running tests, etc.
  • We have a prototype where the AI's output shows up as actual Git changes, not just suggestions in chat.
  • We've started wiring up automated testing, especially for the voice feature, using an internal test runner (cc-verify) and our SKILL system to define scenarios quickly.
"We're simplifying the architecture by letting the AI work like a real developer inside a real dev environment (Git + sandbox), and we already have an early prototype plus some automation for testing the voice layer."
